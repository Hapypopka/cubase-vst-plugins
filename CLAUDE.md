# Cubase VST3 Plugins — инструкции для Claude

## Контекст

Этот репо собирает VST3 плагины для Ильмира (звукорежиссёр, работает в Cubase). Плагины генерируются по запросу через Telegram бот.

**Пользователь — НЕ программист.** Объясняй простыми словами, без упоминания JUCE/CMake/C++/файлов/кода. Когда отвечаешь после правок — одно короткое предложение: что плагин теперь делает.

## Архитектура

- Каждый плагин — в своей папке `plugins/<Name>/`
- Корневой `CMakeLists.txt` автоматически подхватывает все папки в `plugins/*` через `file(GLOB)` — НЕ надо вручную регистрировать новый плагин
- Сборка идёт на GitHub Actions (Windows runner) — НЕ пытайся собирать локально (`cmake`, `dotnet build` и тп). Просто пиши код, пуш делает бот.
- JUCE 7.0.12 (фреймворк) — клонируется в Actions автоматически

## Как создать новый плагин

1. Создай папку `plugins/<Name>/` (имя — английскими, без пробелов: `Distortion`, `SimpleReverb`, `TapeDelay`)
2. Скопируй структуру из `plugins/Gain/` (минимальный пример)
3. **Уникальный `PLUGIN_CODE`** в CMakeLists.txt — ровно 4 символа, не повторяться с другими плагинами в репо
4. **`PLUGIN_MANUFACTURER_CODE`** — оставляй `Ilmr` для всех плагинов
5. Реализуй DSP в `PluginProcessor::processBlock`
6. Реализуй GUI (ручки) в `PluginEditor`

## Как изменить существующий плагин

1. Найди папку в `plugins/<Name>/`
2. Правь `Source/*.cpp` и `Source/*.h`
3. Если нужно добавить параметр (новую ручку):
   - Добавь в `createParameterLayout()` в Processor
   - Добавь `juce::Slider` + `SliderAttachment` в Editor
   - Используй параметр в `processBlock`

## Правила DSP

- **Ручки (параметры)** через `AudioProcessorValueTreeState` — это стандарт
- **Дефолтные значения** — разумные (например drive: 0.5, mix: 1.0, time: 0.25)
- **Range** — типично нормализованные 0..1 для перцентов или физические единицы (Hz, ms, dB)
- **Smoothing** — для параметров что меняются на лету, используй `SmoothedValue` чтобы избежать кликов

## Простые предметные знания (для типовых эффектов)

| Эффект | Минимум что нужно |
|--------|-------------------|
| Дисторшн | Drive (gain до клиппинга), Tone (LP/HP фильтр), Mix (сухой/мокрый) |
| Дилей | Time (мс), Feedback (0-0.95), Mix |
| Ревер | Size (длина хвоста), Damping (затухание ВЧ), Mix |
| Компрессор | Threshold (dB), Ratio, Attack (мс), Release (мс) |
| EQ | Freq (Hz), Q, Gain (dB) на каждой полосе |
| Фильтр | Cutoff (Hz), Resonance |
| Гейт | Threshold, Attack, Release |
| Хорус/Флэнджер | Rate (Hz), Depth, Mix |

## Категории VST3 (для VST3_CATEGORIES в CMakeLists)

- `"Fx Distortion"` — дисторшн, овердрайв, фуз
- `"Fx Delay"` — дилеи, эхо
- `"Fx Reverb"` — ревербы
- `"Fx Dynamics"` — компрессоры, гейты, лимитеры
- `"Fx EQ"` — эквалайзеры
- `"Fx Filter"` — фильтры
- `"Fx Modulation"` — хорус, флэнджер, фазер
- `"Fx Tools"` — гейн, ютилити
- `"Instrument Synth"` — синтезаторы

## После правок

- «Добавил ручку Tone — справа ярче, слева темнее»
- «Сделал ревер с тремя ручками: размер комнаты, затухание ВЧ, и баланс сухой/мокрый»
